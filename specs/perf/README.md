# LSP Performance History

This directory stores historical LSP performance records.
It is only for `scripts/lsp-perf-check.sh`.
Runtime media/profiler benchmarking uses `scripts/perf-media-benchmark.sh` and writes artifacts under `.unity/perf-media/` instead.

## Files

- `lsp-history.jsonl`: append-only execution history generated by `scripts/lsp-perf-check.sh`.

## Metrics

Each run stores:

- Latency metrics (`mean_ms`, `p95_ms`, threshold pass/fail)
- Response size metrics (`response_bytes`, `response_chars`, `response_tokens_o200k`)
- Environment metadata (commit, branch, OS/arch, CLI/LSP versions, timestamp)

Token counting uses the fixed tokenizer `o200k_base`.

## Policy

- `scripts/lsp-perf-check.sh` always executes the full LSP case set.
- `scripts/lsp-perf-check.sh` always appends one JSON line to `lsp-history.jsonl`.
- CI (`.github/workflows/test.yml`) runs the same check on every push/PR and uploads summary/history artifacts.
- History is intended for trend tracking and regression analysis.
